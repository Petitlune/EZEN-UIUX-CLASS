*Redux
 - React.js의 라이브러리
 - 리덕스를 사용하지 않아도 리액트를 사용하는 데에는 문제 없음. 그러나 프론트앤드 개발자로 언제가는 해야됨 

> useReducer() => Redux
useReducer() : 한 컴포넌트 안에 상태변화가 너무 많아서 컴포넌트 바깥으로 빼주어 효율적으로 관리하기 위해 사용했다. 

1. 컴포넌트 안에
const [state, dispatch] = useReducer(reducer, value)
dispatch: 상태변화를 감지하는 촉발함수
이벤트가 발생하게 되면 실행

-> type & action객체

2. 컴포넌트 밖에서 대기하고 있는 reducer함수가 액션 객체를 받아서
실제 실행을 시킴 => 실행된 결과 값을 state에게 업데이트

*독립적인 state data 관리를 시켜주고 싶음!! (data 용량이 큼)
 ==> Redux로 관리해주어야 함. 

*Redux는 동기적인 처리 따라서 비동기처리해주는 tool key를 사용해야함. => Redux Thunk / Redux Saga
