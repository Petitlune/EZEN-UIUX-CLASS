1) JS => 일반객체 > 배열객체
2) 일반객체
     ㄴ>  key, value 값을 가지고 있는 프로퍼티 데이터 체계적으로 관리 보관 사용 / 단점 : for...in => 객체 안에 있는  value값만을 찾아와서 직접적으로 반복문을 돌릴 수 있는 방법XXXX, 객체 안에 구성되어 있는 아이템들은 배열처럼 인덱스 번호를 할당받지 못한다. 

3) 배열객체
     ㄴ> 배열안에 포함되어있는 아이템들이 고유의 인뎃그값을 가지고 있으면,
for문을 활용한 반복문으로 업무의 효율성 극대화
/ 단점 : 객체처럼 key & value 형태로 구성된 체계적인 데이터 관리XXX


======>Solution :일반객체 + 배열객체 장점만 모아서 하나로 만듦
                                                          ㄴ> Map( )
 Map( ) :  key & value 형태로 구성된 아이템을 가질 수 있다, 각 아이템별 인덱스 값 할당받을 수 있다. (for 반복문 사용가능)

Set( )

React.js => Vitual DOM(가상 돔) 사용한다.

Map : 기존 일반객체 & 배열객체의 장점을 모아서 만들어진 Map객체
