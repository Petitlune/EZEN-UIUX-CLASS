> 웹 서버 : 웹과 앱을 사용자에게 제공하는 역할을 수행하는 프로그램 또는 하드웨어
 --  서버는 클라이언트로 요청을 받아 웹페이지의 내용을 전송하여 웹 페이지 구성
 --  웹 서버는 HTML 문서, css, 이미지, js등 파일의 리소스를 제공하여 웹페이지 구성 
 --  사용자로부터 오는 HTTP 요청 받아들이고 적절한 응답
 --  웹 서버는 사용자 접속 이력을 로그로 기록, 모니터링을 통해 최적화
 --  클라이언트 - 서버 모델 === 웹 서버 - 웹 브라우저 사이의 통신 방식을 나타냄 
 => 사용자가 웹페이지를 보기위해 웹 브라우저를 사용하기 때문
 --  클라이언트와 서버 사이의 통신은 요청(request)와 응답(response)를 주고받음
 -- 클라이언트가 웹브라우저를 통해 웹 페이지의 주소를 웹 서버에 요청 => 서버는 요청된 웹 리소스가 저장된 위치를 찾아 해당 파일을 가져옴.  (HTTP 응답 생성: 적절한 응답(코드, 헤더, 본문)) => 생성된 HTTP 응답은 문서 파싱, CSS스타일 적용, js코드 실행으로 페이지 렌더링함.
 --  웹 서버는 클라이언트로부터 동시에 요청받아 처리 => 병렬처리와 동시성 관리 기술이 사용됨
 --  보안 데이터 암호화를 통해 개인정보 보호하며, 로그 기록과 모니터링을 통해 서버 상태 추적하여 문제를 조기에 발견하고 대응할 수 있도록 함. 

 --  web server 종류 : NGINX,  Apache HTTP Server, LiteSpeed(LSWS)
